# 1 웹과 네트워크의 기본에 대해 알아보자
## 1.1 웹은 HTTP 나타난다.
* 웹 브라우저는 웹브라우저 주소 입력란에 지정된 URL에 의지해서
웹서버로부터 리소스라고 불리는 파일등의 정보를 얻고 있는 것입니다.
* 이때, 서버에 의뢰를 하는 웹브라우저 등을 클라이언트라고 부릅니다.
* 이렇게 클라이언트에서 서버까지 일련의 흐름을 결정하고 있는 것은
웹에서 HTTP(HyperText Transfer Protocol)이라 불리는 프로토콜입니다.
* 프로토콜이라는 의미는 약속 입니다.
* 웹은 HTTP라는 약속을 사용한 통신으로 이루어져 있습니다.

## 1.2 HTTP는 이렇게 태어났고 성장했다.
### 웹은 지식공유를 위해 고안되었다.
> www는 연구자들의 지식공유를 지원하기 위해서 제안되었다.

* www를 구성하는 기술
  * 문서기술언어 : html
  * 문서전송프로토콜 : HTTP
  * 문서주소지정방법 : URL
* www는 지금의 웹브라우저, 하이퍼텍스트를 연결할 수 있는 클라이언트 애플리케이션의
명칭으로 사용되어 www 혹은 단순히 Web이라 불리고 있습니다.

### 1.2.2 웹이 성장한 시대
> 대한민국 최초 홈페이지 : http://cair.kaist.ac.kr

* HTML 1.0 초안도 검토되었지만 애매한 부분이 많았기 때문에
초안인 상태로 폐기되었다.

### 1.2.3 진보 안하는 HTTP
* HTTP/0.9 : 정식 사양서는 아님
* HTTP/1.0
* HTTP/1.1 : 현재 가장 많이 사용되는 버전
* 웹 문서 전송 프로토콜로서 등장한 HTTP는 거의 버전이 업그레이드 되지 않았습니다.
* HTTP는 등장한 당시에 주로 텍스트를 전송하기 위한 프로토콜 이었지만,
프로토콜 자체가 상당히 심플해서 여러가지 응용방법을 고려해서 기능이 계속해서
추가되었습니다.

## 1.3 네트워크의 기본은 TCP/IP
* 일반적으로 사용하고 있는 네트워크는 TCP/IP라는 프로토콜에서 움직이고 있습니다.
* HTTP는 그중 하나입니다.

### 1.3.1 TCP/IP는 프로토콜의 집합.
* 컴퓨터와 네트워크 기기가 통신하기 위해서는 규칙을 정할 필요가 있습니다.
* 이러한 규칙을 프로토콜이라고 부릅니다.
* 프로토콜에는 여러가지가 있습니다.
  1. 케이블 규격
  2. IP 주소지정방법
  3. 상대를 찾기위한 방법
  4. 그곳에 도달하는 순서
  5. 웹을 표시하기 위한 순서
* 인터넷과 관련된 프로토콜들을 모은것을 TCP/IP라고한다.

### 1.3.2 계층으로 관리하는 TCP/IP
* TCP/IP는 4계층으로 나뉘어 있습니다.
  1. 애플리케이션 계층
  2. 트랜스포트 계층
  3. 네트워크 계층
  4. 링크 계층
* 계층화 되어 있기 때문에 사양이 변경된 계층만 변경하면 되기 때문에 메리트가 있다.
* 각 계층은 계층이 연결되어 있는 부분만 결정되어 있어, 각 계층의 내부는 자유롭게
설계할 수 있습니다.
* 계층화는 설계를 편하게 한다.

#### 애플리케이션 계층
* 유저에게 제공되는 애플리케이션에서 사용하는 통신의 움직임을 결정하고 있습니다.
* TCP/IP에는 여러 가지의 공통 애플리케이션이 준비되어 잇습니다.
(ex. FTP, DNS)
* HTTP도 이계층에 포함됩니다.

#### 트랜스포트 계층
- 트랜스포트 계층은 애플리케이션 계층에 네트워크로 접속되어 있는 2대의 컴퓨터
사이의 데이터 흐름을 제공합니다.
* 이 계층에는 다른 성질등 가진 TCP와 UDP 두 가지 프로토콜이 있습니다.

#### 네트워크 계층(혹은 인터넷 계층)
* 네트워크 계층은 네트워크 상에서 패킷의 이동을 다룹니다.
* 패킷이란 전송하는 데이터의 최소 단위입니다.
* 이 계층에서는 경로(절차)를 결정하기도 합니다.
* 여러가지 선택지 중 하나의 길을 결정하는 것이 네트워크 계층의 역할입니다.

#### 링크 계층(혹은 데이터 링크계층, 네트워크 인터페이스 계층)
* 네트워크에 접속하는 하드웨어적인 면을 다룹니다.
* 운영체제가 하드웨어를 제어하기 때문에 디바이스 드라이버랑 네트워크 인터페이스 카드
를 포함합니다.
* 케이블 등과 같이 물리적으로 보이는 부분(커넥트 등을 포함한 여러가지 전송매체)도
포함합니다.
* 하드웨어적 측면은 모두 링크계층의 역할입니다.
